# PSLE Ready — 出题、答案、判卷、讲解 规范文档

## 一、核心原则

**宁可少，不可错。** PSLE关系到每个孩子的升学，所有学术内容必须100%准确。

---

## 二、出题规范

### 2.1 题目来源（按优先级）
1. **真实考卷原题** — 名校SA/CA试卷、Prelim试卷、PSLE历年真题
2. **MOE认可教材** — My Pals Are Here!、Shaping Maths等课本例题/习题
3. **基于考纲改编** — 参考以上来源，调整数字但保持题型和考点一致

❌ 禁止：凭空编题、AI生成未验证的题目

### 2.2 每道题必须包含
| 字段 | 说明 | 必填 |
|------|------|------|
| question | 题目文字，清晰完整 | ✅ |
| answer | 标准答案 | ✅ |
| difficulty | basic / intermediate / challenging | ✅ |
| solution | 完整解题步骤 | ✅（推荐） |
| figure | SVG图形（几何题） | 几何题必填 |
| source | 来源（如"Nanyang 2024 SA1 Q15"） | 推荐 |

### 2.3 题目分类要求
- 每个知识点至少覆盖3种难度：Basic（基础）、Intermediate（中等）、Challenging（挑战）
- 题型要覆盖该知识点在PSLE中出现的所有形式（MCQ、填空、应用题、画图题等）
- 每个知识点建议5-8道题

### 2.4 题目文字规范
- 使用新加坡英语表述（如"working"而非"work shown"）
- 数学题中的货币用"$"（新加坡元）
- 单位按MOE标准：cm, m, km, g, kg, ml, L, cm², m², cm³, m³
- 几何题必须配SVG图，不能只靠文字描述

## 三、答案规范

### 3.1 答案分类

| 类型 | 判定方式 | 答案格式要求 | 示例 |
|------|---------|------------|------|
| **纯数字** | 自动判（100%准确） | 只写最终数字+单位 | "115°", "87 cm²", "$60" |
| **数字列表** | 自动判（100%准确） | 用逗号分隔 | "1, 2, 3, 4, 6, 9, 12, 18, 36" |
| **Yes/No** | 按钮选择（100%准确） | 只写"Yes"或"No" | "Yes" |
| **True/False** | 按钮选择（100%准确） | 只写"True"或"False" | "True" |
| **多部分答案** | 自动判（提取数字） | 用括号分段 | "(a) 58 cm² (b) 28 cm" |
| **文字解释** | 不自动判，展示标准答案 | 写完整标准答案 | "The ball slows down because..." |

### 3.2 关键规则
- **答案字段(answer)** 只写最终答案，不写过程
- **解题步骤(solution)** 写完整过程，用\n换行
- 数字答案不要带多余文字（❌ "The answer is 115°" ✅ "115°"）
- 列表答案用英文逗号+空格分隔
- 每个答案必须人工验算确认

### 3.3 验算检查清单
每道题录入前必须完成：
- [ ] 手动计算答案，确认正确
- [ ] 检查solution步骤是否完整
- [ ] 检查solution每一步计算是否正确
- [ ] 检查answer字段与solution最终结果是否一致
- [ ] 检查单位是否正确

## 四、判卷规范

### 4.1 自动判卷（仅限100%确定的场景）

**可以自动判的：**
```
• 纯数字答案 → 提取数字比较
  用户输入"115" 或 "115°" 或 "115 degrees" → 提取115 → 与答案115比较
  
• 数字列表 → 提取所有数字，支持无序比较
  答案"1, 2, 3, 4, 6, 9, 12, 18, 36"
  用户输入"1,2,3,4,6,9,12,18,36" → 提取数字集合 → 比较 → ✅

• Yes/No → 按钮点选，不存在误判

• True/False → 按钮点选，不存在误判

• 选择题（options字段） → 按钮点选，精确匹配
  如 options: ["Acute", "Obtuse", "Reflex"]
  适用于：答案固定、可穷举选项的文字答案题
```

**不能自动判的：**
```
• 文字解释题 — "Explain why..."
• 应用题解题过程 — 只能判最终数字答案
• Science OEQ — 开放式回答
• English/Chinese 阅读理解 — 需要语义理解
• 任何需要"理解"才能判断的答案
```

### 4.2 数字提取逻辑
```javascript
// 提取完整数字（含小数），用parseFloat标准化去除尾部零
extractNumbers("$38.70") → ["38.7"]
extractNumbers("38.7")   → ["38.7"]   // 38.7 = 38.70 ✅

// 处理逗号分隔的大数字（如40,215 = 40215）
extractNumbers("40,215, 40,521") → ["40215", "40521"]

// 处理带单位的数字
extractNumbers("87 cm²") → ["87"]
extractNumbers("$1,700") → ["1700"]
extractNumbers("47.5°C") → ["47.5"]

// 比较规则：
// 1. 提取所有数字（小数作为完整数字，不拆分）
// 2. 用parseFloat标准化（38.70 → 38.7，去除尾部零）
// 3. 按顺序比较（排列题）
// 4. 如果顺序不匹配，按大小排序后比较（列举题）
```

**等价判定原则：** 学生输入与标准答案在数学意义上相等即为正确：
- `38.7` = `38.70` = `$38.70` ✅
- `7000` = `7,000` ✅
- `115` = `115°` ✅
- 不带/带单位符号（$, °, cm, kg等）均可

### 4.3 判卷结果展示
```
✅ 答对：
   绿色背景 + "Correct! Well done!"
   可以查看标准解法

❌ 答错：
   红色背景 + "Not quite."
   [View Answer] — 查看标准答案和解法
   [Ask Tutor] — 弹出Sproutie讲解

📖 自评模式（文字题）：
   不显示对错
   [View Answer] — 查看标准答案
   学生自己对比
   [Don't understand? Ask Tutor]
```

### 4.4 绝对禁止
- ❌ 对不确定的答案显示✅或❌
- ❌ 用模糊匹配判文字答案
- ❌ 把正确答案判为错误（最严重的错误）
- ❌ 把错误答案判为正确

## 五、讲解规范

### 5.1 解题步骤（Solution）规范

**Math解题步骤要求：**
- 必须使用MOE认可的12种heuristics方法
- 应用题必须写"statement"（答案句子），如"Tom had 72 apples at first."
- 每一步要写清楚在做什么
- 步骤之间用换行分隔

示例：
```
Fraction of money spent = 1/4 + 2/5
= 5/20 + 8/20 = 13/20

Fraction left = 1 − 13/20 = 7/20

7/20 of his money = $21
1/20 of his money = $21 ÷ 7 = $3
20/20 of his money = $3 × 20 = $60

Ali had $60 at first.
```

**Science OEQ解题步骤要求：**
- 使用CER框架：Claim（结论）→ Evidence（证据）→ Reasoning（推理）
- 标出得分关键词
- 按SEAB marking scheme格式

**English解题步骤要求：**
- Synthesis & Transformation：写出转换规则
- Comprehension OEQ：指出答案在passage哪里找到
- Situational Writing：标注content points

### 5.2 Tutor引导讲解（Sproutie）
- 苏格拉底式：从不直接告诉答案
- 一步步引导学生思考
- 使用MOE标准教学方法
- 鼓励而非批评
- 详见 `/src/lib/prompts.ts` 中各科目的system prompt

### 5.3 Worked Examples（例题）规范
- 每个知识点至少1-2个worked example
- 从简单到复杂
- 展示完整解题过程
- 标注使用的方法（如"Model Drawing"、"Before-After"）

## 六、质量保障流程

### 6.1 内容录入流程
```
1. 确定知识点（对应MOE大纲章节）
2. 选择/改编题目（真实考卷来源）
3. 人工解题并验算答案
4. 编写完整solution步骤
5. 检查答案格式是否符合自动判卷要求
6. 几何题绘制SVG图形
7. 设定难度级别
8. 最终检查：answer字段 = solution最后一步结果
9. 录入系统
```

### 6.2 答案核查核算验证

每道题在发布前必须通过以下三级验证：

**第一级：自算验证**
```
录入者独立计算答案，不参考任何来源
→ 与录入的answer字段对比
→ 不一致 → 重新计算，找出差异原因
→ 一致 → 进入第二级
```

**第二级：交叉验证**
```
第二人（或AI辅助）独立计算同一道题
→ 与第一级结果对比
→ 不一致 → 两人讨论，确定正确答案
→ 一致 → 进入第三级
```

**第三级：回溯验证**
```
用正确答案代入题目条件，反向验证是否成立
例：
  题目：A square has a perimeter of 36 cm. Find its area.
  答案：81 cm²
  回溯：area = 81 → side = 9 → perimeter = 9×4 = 36 ✅ 成立
```

**特殊题型验证：**

| 题型 | 额外验证步骤 |
|------|------------|
| 分数/百分比 | 检查最简分数、百分比换算是否正确 |
| 单位换算 | 检查单位是否一致，有无漏转换 |
| 几何计算 | 用不同方法（公式法 vs 分割法）交叉验算 |
| 应用题 | 检查statement句子是否准确回答了问题 |
| 多步骤题 | 逐步验算每一步的中间结果 |
| 比例/Ratio | 用回原始数据验证比例关系是否成立 |

**验证记录模板：**
```
题目ID: p4c12-3
题目: A rectangular garden is 20m long and 15m wide...
录入答案: 124 m²
自算结果: 300 - 176 = 124 ✅
回溯验证: 内部 16×11=176, 外部 20×15=300, 差=124 ✅
状态: 已验证
验证人: [name]
验证日期: [date]
```

### 6.3 错误处理
- 每道题旁有 ⚑ 报告按钮（淡色，不明显）
- 用户报告后进入审核队列
- 确认错误 → 立即修正
- 错误类型记录：答案错误 / 解法错误 / 题目不清 / 难度不当

### 6.3 定期审核
- 每学期对照MOE最新大纲检查一次
- 新版大纲发布后30天内完成内容更新
- 记录变更日志

## 七、各科特殊要求

### Math
- 优先使用Model Method（模型画图法）
- 应用题solution必须包含statement（答案句）
- 几何题必须有SVG图
- 方法必须是MOE 12种heuristics之一
- 2026变化：Speed已移除、Ratio移到P6、Algebra新增P6

### Science
- OEQ使用CER框架
- 标注得分关键词
- 实验题标注变量（测试变量/控制变量/结果变量）
- 5大主题：Diversity, Cycles, Systems, Energy, Interactions

### English
- **新加坡使用British English体系**，所有拼写、语法、用词必须遵循British English标准
  - 拼写：colour (非color), favourite (非favorite), centre (非center), organise (非organize), behaviour (非behavior), programme (非program)
  - 语法：collective nouns视情况可用复数 (e.g., "The team are ready")，"have got" (非"have gotten")
  - 日期格式：DD/MM/YYYY (e.g., 11 July 2026, 非July 11, 2026)
  - 标点：引号用单引号 'like this' 为主，双引号用于引号中的引号
  - 词汇：lift (非elevator), boot (非trunk), queue (非line), rubbish (非trash/garbage), maths (非math)
- Comprehension按passage内容答题
- Synthesis & Transformation注明转换规则
- Situational Writing标注content points（2025新增underlined推断点）
- Grammar题注明语法规则

### Chinese
- 可中英混合讲解（适配新加坡学生）
- 阅读理解标注答案出处
- 作文按四段式结构
- 口试按5W1H框架
