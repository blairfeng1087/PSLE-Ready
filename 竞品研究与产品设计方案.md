### 1.3 市场空白与机会

**现有产品的共同缺陷：**
1. **Tutorly/Klara** — 纯文字聊天框，无拍照识题、无错题本、无家长看板
2. **Geniebook** — 太贵，年均S$1,852/科
3. **小猿AI** — 中国内容，不适配新加坡MOE课纲和PSLE
4. **LearnBuddy** — 有苏格拉底+家长看板，但无错题本和拍照功能

**我们的差异化定位：**
> 唯一一款同时具备「拍照识题 + 苏格拉底AI辅导 + 智能错题本」的新加坡PSLE专属学习工具

## 二、用户画像 (Persona)

### Persona 1：学生 — Emily, P5女生

- **年龄**: 11岁，Primary 5
- **科目**: Math, Science, English, Chinese (Higher Chinese)
- **学习习惯**: 每天放学后做作业1.5-2小时，周末有补习班
- **痛点**:
  - 数学应用题(Problem Sums)经常卡住，不知道从哪一步开始
  - 做错的题过几天又忘了，同类型错误反复犯
  - 科学OEQ(开放式问题)不知道怎么组织答案结构
  - 不敢问老师/补习老师，怕被觉得笨
- **期望**: 有人能耐心一步步教我，不要直接告诉答案（因为考试时没人帮忙）
- **设备**: iPad / 妈妈的iPhone
- **使用场景**: 做完作业后复习错题、考试前刷题

### Persona 2：家长 — Mrs. Tan, Emily的妈妈

- **年龄**: 40岁，职业女性
- **痛点**:
  - 下班后没精力辅导，尤其数学Higher Chinese自己也不会
  - 补习费太贵（S$150-300/月/科），4科全补承受不起
  - 不知道孩子到底哪个知识点薄弱
  - 想帮孩子整理错题本但没时间手抄
  - 不想让孩子依赖AI直接抄答案
- **期望**: 一个能代替补习老师的工具，价格合理，能看到孩子的学习进展
- **核心决策因素**: 价格 < S$50/月、能看到效果、不会让孩子偷懒

## 三、Phase 1 核心功能设计

### 功能A：AI老师（苏格拉底式引导）

**用户故事**: Emily拍了一道数学题，AI不告诉她答案，而是一步步引导她自己做出来。

**交互流程**:
```
Emily: [拍照上传一道Problem Sum]

AI: "我看到这道题了。我们一起来解决它。
     先告诉我，这道题在问什么？你能用自己的话说一下吗？"

Emily: "Dora和Sarah一开始有一样多的钱"

AI: "很好！你找到了第一个关键信息。
     那接下来发生了什么变化呢？再读一遍题目看看。"

Emily: "Dora花了$450，Sarah从祖父母那收到$350"

AI: "对了！那现在她们的钱还一样多吗？
     如果一开始都是 1u，现在各是多少呢？试试用model画出来。"

Emily: "D是 1u - 450，S是 1u + 350"

AI: "太棒了！题目还说了什么关于她们最后的钱的关系？"

Emily: "Sarah最后是Dora的3倍"

AI: "所以 1u + 350 = 3 × (1u - 450)。你能试着解这个方程吗？"

[继续引导直到Emily自己算出答案]

AI: "恭喜你自己做出来了！🎉 
     这道题的关键是：先画model，再列方程。
     要不要把它加到错题本里以后复习？"
```

**AI提示词设计原则** (参考Khanmigo):
1. 永远不直接给出答案或解题步骤
2. 每次只推进一步，等学生回应
3. 如果学生卡住超过2次，给一个小提示（不是答案）
4. 用鼓励性语气，建立自信
5. 做完后总结方法论，而不是答案本身
6. 针对新加坡数学的Model Method特别优化

**技术实现**: Claude API + 精心设计的System Prompt，按科目切换不同教学策略

---

### 功能B：智能错题本

**用户故事**: Emily做完一套数学试卷，拍照上传，AI自动识别题目、擦除笔迹、标记错题并归类。

**交互流程**:
```
[拍照上传试卷页面]
        ↓
[AI识别] → 识别出5道题
        ↓
[展示识别结果] → 用户确认/修正
        ↓
[标记错题] → 用户勾选哪些做错了
        ↓
[错因分析] → AI分析："这道题考的是Ratio，你的错误在于没有找到unchanged quantity"
        ↓
[自动归类] → Math > Ratio > Finding Unchanged Quantity
        ↓
[错题本] → 按学科/知识点/日期 三维浏览
        ↓
[导出] → 选择多题 → 生成PDF → 分享/打印
```

**错题本数据结构**:
- 学科 (Math / Science / English / Chinese)
- 知识模块 (如Math: Fractions, Ratio, Percentage, Speed, etc.)
- 题型 (如Math: Problem Sum, MCQ, Short Answer)
- 难度 (P3-P6 / PSLE难度)
- 错因标签 (概念不清 / 计算错误 / 审题不仔细 / 方法选错)
- 原题图片 + 清洁版图片
- AI错因分析文字
- 日期
- 复习状态 (未复习 / 已复习 / 已掌握)

**关键体验细节** (参考小猿AI最佳实践):
- OCR识别后过一遍AI校正，降低识别错误率
- 用户可以一键标记"这道我做错了"
- 错题可按知识点导出，考前复习精准命中
- 支持"相似题推荐"（Phase 2）

## 六、技术栈建议

| 组件 | 技术选择 | 原因 |
|------|---------|------|
| App | Flutter (iOS) | 已有基础，跨平台 |
| AI对话 | Claude API (Sonnet) | 苏格拉底引导效果最佳，性价比高 |
| 题目识别 | Claude Vision API | 能区分印刷/手写，提取题目 |
| 笔迹擦除 | 有道智云API | 已集成，效果最佳 |
| 图片裁剪 | 本地OpenCV预处理 | 边缘检测+透视矫正 |
| 数据存储 | 本地SQLite + 云端备份(Phase 2) | 简单可靠 |
| PDF生成 | Flutter pdf包 | 客户端生成，已实现 |
