# 日历打卡逻辑

## 数据结构

```typescript
practiceLog: {
  "2026-07-01": 2,  // 2=全部完成
  "2026-07-02": 1,  // 1=部分完成
  "2026-07-03": 0,  // 0=未打卡（有任务但没做）
  // 没有记录的过去日期 = 未打卡
  // 未来日期 = 不显示状态
}
```

## 打卡状态判定

每天系统自动生成4个练习任务。打卡状态根据完成情况自动计算：

| 完成情况 | 状态值 | 日历显示 |
|---------|-------|---------|
| 4/4 全部完成 | 2 (done) | 🟢 深绿色方块 |
| 1-3/4 部分完成 | 1 (partial) | 🟢 浅绿色方块 |
| 0/4 一题没做 | 0 (missed) | 🔴 淡红色方块 |
| 今天 | 当前状态 | 🟢 深绿+白色描边 |
| 未来日期 | 无 | ⬜ 灰色方块 |

## 打卡触发时机

```
用户点击 "Start" 做任务
    → 完成一个任务
    → completeTask(taskId) 
    → 更新 todayTasks 中该任务 done=true
    → 计算当天完成比例
    → 更新 practiceLog[今天日期] = 1或2
    → 重新计算 streak
    → 日历热力图自动刷新
```

## Streak（连续打卡）计算

```
从今天往前回溯：
  - 今天如果还没打卡，从昨天开始算
  - 连续有 status>=1 的天数即为 streak
  - 遇到 status=0 或无记录则中断

示例：
  7/9: done(2)  ← 今天
  7/8: done(2)
  7/7: done(2)
  7/6: done(2)
  7/5: done(2)
  7/4: partial(1) ← 也算打卡
  7/3: done(2)
  7/2: done(2)
  7/1: partial(1)
  → streak = 9 天
```

## 日历月份切换

- 默认显示当前月
- 左右箭头切换月份
- 每月从周日开始排列（Singapore标准）
- 月首前的空位留空
- 底部显示当月打卡率：`done_days / past_days`

## 与每日练习的联动

```
Daily Practice 页面：
  ┌──────────────────────┐
  │ 📐 Math · Ratio  [Start +60XP] │  ← 点击Start
  └──────────────────────┘
            ↓
  做题流程（Phase 2接入AI）
            ↓
  完成 → completeTask("t1")
            ↓
  ┌──────────────────────┐
  │ 📐 Math · Ratio  ✅ +60 │  ← 变为已完成状态
  └──────────────────────┘
            ↓
  practiceLog["2026-07-10"] = 1 (partial)
            ↓
  日历 7/10 格子变为浅绿色
            ↓
  再完成剩余3个任务
            ↓
  practiceLog["2026-07-10"] = 2 (done)
            ↓
  日历 7/10 格子变为深绿色
  streak + 1
```
